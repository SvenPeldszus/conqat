Base data structures for storing unit test results. Test are organized in the following hierarchical structure:

  1. TestRoot: Root node to summarize test runs of one or different tools.
  2. TestRun: Resembles a single execution of a test tool, e.g. for one binary.
  3. TestContainer: Meta container that holds the the test results. Usually container correspond to single test classes.
  4. TestMethod: The actual executed test method. This node holds the test result.
